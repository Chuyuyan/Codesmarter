# 🔧 修复标签页消失问题

## 🐛 问题

重新登录后，上传文件的标签页（三个按钮）不见了。

---

## ✅ 解决方案

### **第一步：检查您访问的 URL**

**打开浏览器，查看地址栏中的 URL：**

1. **如果是：** `http://127.0.0.1:5050/` 
   - ❌ 这是首页（landing page），没有上传功能
   - ✅ 需要访问：`http://127.0.0.1:5050/dashboard`

2. **如果是：** `http://127.0.0.1:5050/dashboard`
   - ✅ 这是正确的页面
   - 但如果还是看不到标签页，继续第二步

---

### **第二步：访问正确的页面**

**在浏览器地址栏输入：**
```
http://127.0.0.1:5050/dashboard
```

然后按 `Enter`。

---

### **第三步：硬刷新浏览器**

访问 `/dashboard` 后，按 **`Ctrl + F5`** (Windows) 或 **`Cmd + Shift + R`** (Mac) 硬刷新。

---

### **第四步：清除浏览器缓存（如果还不行）**

1. 按 `F12` 打开开发者工具
2. **右键点击**浏览器的刷新按钮
3. 选择 **"清空缓存并硬性重新加载"** 或 **"Empty Cache and Hard Reload"**

或者：

1. 按 `Ctrl + Shift + Delete` (Windows) 或 `Cmd + Shift + Delete` (Mac)
2. 选择 **"缓存的图像和文件"**
3. 点击 **"清除数据"**

---

## 🎯 修复后应该看到

在 `/dashboard` 页面，您应该看到：

1. **三个标签页按钮**：
   - 📤 Upload Folder (ZIP) ← 默认选中
   - 🔗 Git URL
   - 📁 Enter Path

2. **上传标签页内容**（默认显示）：
   - 文件选择器
   - Repository Name 输入框
   - "Upload and Index" 按钮

---

## 🔍 快速检查清单

- [ ] 确认访问的是 `/dashboard` 而不是 `/`
- [ ] 硬刷新浏览器 (`Ctrl + F5`)
- [ ] 检查浏览器控制台是否有错误（按 `F12`）
- [ ] 检查 Network 标签页，确认 `dashboard.html` 已加载

---

## 💡 为什么会出现这个问题？

1. **路由变更**：现在 `/` 指向 landing page（营销页面），`/dashboard` 才是应用页面
2. **浏览器缓存**：浏览器可能缓存了旧版本的 HTML 文件
3. **访问错误页面**：登录后可能没有正确重定向到 `/dashboard`

---

## 🚀 快速修复步骤

1. ✅ **访问**：`http://127.0.0.1:5050/dashboard`
2. ✅ **硬刷新**：`Ctrl + F5`
3. ✅ **应该能看到标签页了！**

---

如果按照以上步骤操作后仍然看不到标签页，请提供：
- 浏览器控制台错误信息（按 F12）
- 浏览器地址栏显示的完整 URL
- Network 标签页中 `dashboard.html` 的响应内容
